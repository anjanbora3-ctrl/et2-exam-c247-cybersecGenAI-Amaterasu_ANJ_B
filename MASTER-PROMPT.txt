SCENARIO BASED ASSESSMENT: ATTACK SURFACE MONITORING AND SECURITY AUDIT SYSTEM

MAIN PROMPT

CREATE AN ATTACK SURFACE MONITORING SYSTEM THAT CONTINUOUSLY AUDITS INFRASTRUCTURE CONFIGURATIONS AND DETECTS SECURITY RISK.

DEVELOP A SECURITY AUDITING SYSTEM THAT MONITORS INFRASTRUCTURE CONFIGURATIONS AND IDENTIFIES ATTACK SURFACE RISKS.

THE SYSTEM SHOULD PERFORM THE FOLLOWING TASKS.

INFRASTRUCTURE SECURITY MONITORING

USING TERRAFORM CONFIGURATIONS, ANALYZE INFRASTRUCTURE SETTINGS AND DETECT:

1. PUBLICLY EXPOSED SERVERS
2. INSECURE STORAGE CONFIGURATIONS
3. OPEN NETWORK PORTS
4. WEAK SECURITY GROUPS

DEPENDENCY AND APPLICATION SECURITY

USING SNYK OR OPENSOURCE CODE SCANNING TOOLS, ANALYZE APPLICATIONS TO DETECT:
1. VULNERABLE DEPENDENCIES
2. OUTDATED LIBRARIES
3. HIGH-RISK PACKAGES

AUTOMATED SECURITY CHECKS
USING GITHUB ACTIONS/ GITLAB CI/ JENKINS, DESIGN A WORKFLOW THAT:
1. AUTOMATICALLY RUNS SECURITY CHECKS
2. GENERATE AUDIT REPORTS
3. FLAGS RISKY CONFIGURATIONS

SECURITY INSIGHTS
THE SYSTEM SHOULD PROVIDE INFORMATION SUCH AS:
1. NUMBER OF EXPOSED SERVICES
2. HIGH-RISK DEPENDENCIES
3. INSECURE INFRASTRUCTURE CONFIGURATIONS
4. AUDIT SUMMARY OF SECURITY POSTURE

DATASET/ INPUT TO BE USED:
THE DATASET TO BE USED IS IN THE FILE NAMED "ET2-DATASET-MAIN.xlsx". IT IS IN THE CURRENT FOLDER.

DELIVERABLES
FINAL REPORT (4-5 PAGES PDF): REPORT MUST INCLUDE THE FOLLOWING SECTIONS
A. OVERVIEW
EXPLAIN:
   * ATTACK SURFACE RISKS
   * OBJECTIVE OF THE MONITORING SYSTEM

B. DATASET/CONFIGURATION EXPLAINATION
DESCRIBE:
   * INFRASTRUCTURE CONFIGURATION ANALYZED
   * DEPENDENCY DATA

C. METHOD USED
EXPLAIN:
  * HOW TERRAFORM CONFIGURATIONS WERE ANALYZED
  * HOW VULNERABILITIES WERE DETECTED
  * HOW CI/CD SECURITY CHECKS WERE IMPLEMENTED

FINDINGS
  * HIGHLIGHT THE MAJOR SECURITY RISKS IDENTIFIED.

RECOMMENDATIONS
  * PROVIDE 2-3 RECOMMENDATIONS TO IMPROVE INFRASTRUCTURE SECURITY
